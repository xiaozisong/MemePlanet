## ADR-001 前端 App 选型：React Native + Expo

- **状态**：✅ 已定稿（2026-07-07）
- **背景**：个人开发者全栈，需 iOS+Android+Web，预算极有限。
- **选择**：MVP 用 React Native + Expo（managed workflow + EAS Build/Submit）；v2.0 视情况评估 Flutter 重写。
- **理由**：全栈 TS 复用最大化（与 NestJS/Next.js 共享 `packages/shared`）；Expo Update 热更新绕过商店审核；AI/视频插件生态成熟（react-native-video / onnxruntime / ffmpeg）；Vercel AI SDK 在 JS 侧最丰富。Flutter 唯一胜出的 PK 动画质感可用 reanimated + skia 补齐。
- **影响**：放弃微信小程序首发；App 业务逻辑可与 Web/后端共享类型与 API client。
- **放弃项**：Taro 多端方案、Flutter MVP。

## ADR-002 后端框架：NestJS（TS），PK 服务未来按需迁 Go

- **状态**：✅ 已定稿（2026-07-07）
- **背景**：个人开发者，DAU 1w 量级，AI 编排生态重要。
- **选择**：MVP 全 NestJS(TS) 模块化单体，15 Module；仅当 PK 投票峰值到 DAU 10w+ 且 Redis 优化到极限时，单独把 PK 服务抽出来用 Go 重写。
- **理由**：全栈同语言是个人开发者最大杠杆；NestJS + Redis + PG 扛得住 DAU 1w；Go 的优势仅在 PK 高并发单点，可用 Redis ZSet + BullMQ 削峰解决；Java/Spring 在 2C2G 跑不动。
- **影响**：后端/前端/共享层全部 TS，类型可复用。

## ADR-003 AI 造梗 Agent：MVP 引入，Pro 专属，3 步精简版

- **状态**：✅ 已定稿（2026-07-07）
- **背景**：传统单次 prompt 质量一般，Agent 多步推理可显著提升梗质量，但成本×3-5、延迟 15-30s。
- **选择**：MVP 引入造梗 Agent，**仅 Pro 会员**，3 步精简版（RAG 检索神梗 top5 → 生成 3 候选 → LLM 自评选优）。免费用户保持单次 prompt。10 次/日硬配额 + 日预算 ¥80 熔断降级为单次 prompt。v1.5 全用户开放 + 视频脚本生成步。
- **理由**：Pro 专属控成本 + 引流付费；3 步上限避免串联昂贵视频模型；异步任务化（BullMQ + 推送通知）解决延迟体验；用 Vercel AI SDK + DeepSeek V3（支持 function calling）。
- **影响**：Pro 会员成为 Agent 承载前提，Pro 必须在 v1.0 上线；新增 agent_jobs / agent_steps 表；新增 RAG 知识库（pgvector 神梗向量化）。

## ADR-004 AI 模型来源（不自研，全接入第三方）

- **状态**：✅ 已定稿（2026-07-07）
- **选择**：
  - LLM：DeepSeek V3 主力（¥1/M tokens）+ GLM-4 Flash 兜底（完全免费）+ Qwen 备选
  - 图像：SiliconFlow FLUX-schnell 主力 + 通义万相 fallback
  - 视频：**字节豆包 Seedance 2.0 mini（MVP 主力）** + Seedance 2.0 标准版（Pro 高端）+ SiliconFlow/即梦 fallback
  - TTS：火山引擎
- **理由**：个人开发者不自研模型；DeepSeek 极便宜中文好；豆包质量好字节生态合规友好（已备案）；多 provider 轮换免费额度降本。
- **影响**：需自建 AI 编排层（AIOrchModule）抽象 provider，便于切换与降级。

## ADR-005 视频生成配额：免费 1次/周·Pro 3次/日

- **状态**：✅ 已定稿（2026-07-07）
- **背景**：豆包 Seedance ¥1/秒（5s≈¥5），DAU 1w 月成本可能 ¥1w+，远超个人预算。
- **选择**：免费用户 1 次/周（需任务解锁）+ 默认图片+TTS+Ken Burns 兜底（覆盖率 40-60%）；Pro 3 次/日（v1.1 从 10 次下调，对齐豆包成本现实）。
- **理由**：豆包质量好但贵 5-10 倍，必须严格配额 + 兜底才能控成本；Pro 3 次/日是 Pro 收入（¥18×500=¥9000）能覆盖的增量成本上限。
- **影响**：免费用户体验部分降级为"伪视频"；需日预算 ¥200 硬熔断；MVP 必须灰度 DAU ≤ 500~2000。

## ADR-006 Pro 会员定价：¥18/月 + 视频按次包

- **状态**：✅ 已定稿（2026-07-07）
- **选择**：Pro 月费 ¥18 起 + 视频按次包（¥9.9/10次、¥29.9/50次），MVP 即开微信支付。年费/季费后续视转化数据再定。
- **理由**：¥18 × 500 Pro = ¥9000/月，可覆盖 DAU 5000 规模的 Agent + 视频增量成本；视频按次包对冲豆包高成本。
- **影响**：Pro 从 v1.5 提前到 v1.0；需接微信支付 + 订单/订阅/支付表；Pro 是 Agent 与视频高配额的承载前提。

## ADR-007 数据部署：主数据国内自建 PG + Supabase 仅 Auth/Realtime

- **状态**：✅ 已定稿（2026-07-07）
- **选择**：主数据放国内自建 PostgreSQL 16（含 pgvector），跑在轻量云 2C2G；Supabase 仅做 Auth + Realtime；用户表通过 Supabase Webhook 同步到国内 PG（仅必要字段）。
- **理由**：国内访问快、合规友好、无流量限制；Supabase 免费档 Auth/Realtime 够用；主数据自建可控。
- **影响**：需 Webhook 同步机制（MVP 可简化为轮询）；Supabase 项目部署新加坡/东京区域 + Cloudflare 边缘代理。

## ADR-008 IM 方案：Supabase Realtime（私聊）+ 自建 WebSocket（军团群聊）

- **状态**：✅ 已定稿（2026-07-07）
- **背景**：调研 OpenIM（成熟但需 4C8G+¥150/月+5 中间件，超规格）、Centrifugo（仅推送层）、Matrix（过重）、商业 IM 云（¥1000+/月超预算）。
- **选择**：私聊 + 系统通知用 Supabase Realtime（免费档 200w 连接）；军团群聊自建 NestJS Gateway + Redis pubsub + PG 持久化（¥0 增量，~1-2 周开发量）；v1.5 评估 Centrifugo，v2.0 评估 OpenIM。
- **理由**：个人开发者 MVP 阶段 ¥0 增量成本 + 复用现有栈 + 可控性最强；群聊需细粒度风控（频控/敏感词），自建更可控。
- **影响**：需自己实现心跳/重连/离线消息/@/引用/已读/历史分页；单机连接上限 ~1w，MVP 灰度够用。

## ADR-010 MVP 灰度策略：DAU ≤ 500~2000 邀请制

- **状态**：✅ 已定稿（2026-07-07）
- **选择**：MVP 上线前 2 周邀请制灰度，DAU ≤ 500 验证产品闭环（豆包成本可控在 ¥3000/月内）；逐步放开到 DAU 2000；全量视成本看板决定。

## ADR-011 视频兜底体验：图片+TTS+Ken Burns 伪视频

- **状态**：✅ 已定稿（2026-07-07）
- **选择**：免费用户默认走"图片 + TTS + Ken Burns 缩放动效"兜底方案（ffmpeg 合成，成本≈图片费+TTS费，~¥0.2/次），覆盖率 40-60%；真视频限 1 次/周。
- **理由**：豆包真视频 ¥5/次 vs 兜底 ¥0.2/次，差 25 倍；兜底覆盖率是豆包成本可控的关键前提；体验接近真视频。
- **影响**：需 ffmpeg 合成链路；免费用户体验部分降级，需产品文案引导。

## ADR-012 军团数量上限：MVP 限 1000 个

- **状态**：✅ 已定稿（2026-07-07）
- **选择**：MVP 全网军团数量上限 1000 个，控制运营成本与冷启动内容质量。
- **理由**：个人开发者运营精力有限；1000 个军团足够验证 PK 玩法；避免冷启动期低质军团泛滥。
