### 1.3 录制注意事项

- 录制前清除模拟器/真机的数据（全新启动）
- 确保已用真实手机号登录（不能是模拟器号码）
- 展示空状态 → 数据加载 → 完整内容的全过程
- 拍摄时不要出现个人信息、敏感词
- 背景音乐可选（建议用无版权轻音乐）

| **合计** | — | **35.8** | **~31.8** | **~89%** | — |

### 2.2 主要成就

1. **完整闭环上线**：从手机号登录 → 文本造梗 → 梗卡发布（含机审）→ Feed 推荐 → 评分评论 的端到端流程全部在 RN 上跑通。
2. **合规材料启动**：AIGC 备案材料（12 章节 ≥80%）、隐私政策、用户协议、未成年人保护设计稿全部完成草稿。
3. **全栈质量**：typecheck=0 / lint=0 贯穿所有模块，166 测试用例 100% 通过。
4. **Harness 信噪比优化**：340 技能瘦身到 107，路由分类层建立。

### 2.3 待外部 key 阻塞项

| 事项 | 阻塞任务 | 预计完成 | 影响 |
| --- | --- | --- | --- |
| DeepSeek V3 API key | T1.10 LLM Adapter 真实调用 | M1 上线前 | 造梗目前用 Mock，需真实 LLM |
| GLM-4-Flash API key | T1.10 LLM Adapter 兜底 | M1 上线前 | 同上，降级兜底 |
| 阿里云内容安全 key | T2.10 接入文本/图片审核 | M1 上线前 | 目前用 DFA 基础过滤 |
| 阿里云短信签名 | T1.3 真实短信下发 | M1 上线前 | 目前 OTP 仅控制台打印 |

### 2.4 关键 Lessons Learned

1. **外部 Key 是最大风险**：最晚应在 S0 提交申请，AI/短信/内容安全全部需要审批时间。
2. **RN 开发效率**：T3.7-T3.14 8 个任务实际耗时 ~9 人日（不含设计系统），比计划 6 人日高 50%。RN 的 Expo Router + NativeWind 在快速原型时很快，但类型对齐（camelCase vs snake_case）和依赖安装会有坑。
3. **个人开发者产能上限**：35.8 人日计划超出 4 周 * 5.5 天 = 22 天有效产能。通过砍项和加班可做到 ~32 天，但不可持续。
4. **Drizzle vs raw SQL 并存**：目前部分模块用 Drizzle schema（5 表），大部分用 schema.sql 直跑。建议 M2 统一到 Drizzle migrations。

### 3.3 M2 砍项建议

若 M2 同样只有 22 天有效产能，建议保留 P0 18 项（~23 人日），P1 按时间剩余做，P2 全延 M3：

1. **P0 必做**（~23 人日）：LLM 真实 + 视频造梗 + 军团 + PK + IM + Pro 会员微信支付 + Admin 审核 UI

3. **P2 延 M3**（~2.5 人日）：E2E 测试、部署 runbook
